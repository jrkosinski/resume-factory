## Context

The client had existing prediction-market strategies that were underperforming, high-risk, and producing unreliable P&L. The project was created to provide a more disciplined trading and research system: one that could ingest historical data, test strategies before forward deployment, compare performance, tune parameters, and cautiously move successful strategies into live trading.

The system now supports production real-money execution and is currently in a small-profit phase, with scale increasing cautiously as strategies are tuned and validated.

## Strategy Framework

Strategies are represented as TypeScript classes that conform to a shared strategy interface. Each strategy has its own configuration/options, allowing the system to run, backtest, compare, optimize, and deploy different strategy families through a common framework.

Successful strategies include:

* Avellaneda-Stoikov market making
* OFI market making
* Modified Dutch Arbitrage spread-capture for ranging markets
* Additional strategy variants

This is a significant architecture detail because the system is designed as a reusable strategy platform, not a collection of isolated trading scripts.

Possible future phrasing:

> Designed a modular TypeScript strategy framework where each strategy is implemented as a class behind a shared interface, enabling consistent backtesting, optimization, comparison, and live execution across multiple strategy families.

## Backtesting Engine

The backtesting engine supports:

* Historical simulation
* Train/test split
* Strategy-specific options
* Side-by-side strategy comparison
* Saved backtest runs
* Saved optimization runs
* Bayesian optimization through Optuna
* Detailed metrics and diagnostics

Metrics include:

* Calmar ratio
* Profit Factor
* Raw P&L
* Max drawdown
* Inventory risk
* Win rate
* ROI
* Expectancy
* Sharpe ratio

The backtester models:

* Fees
* Slippage
* Latency

It does not currently model:

* Partial fills
* Order-book depth

## AI / Agent Layer

The system includes embedded LLM-wrapper agents. They are intentionally lightweight rather than LangChain-based because the current complexity does not justify a heavier orchestration framework.

Agent capabilities include:

* Trading analysis
* Backtesting analysis
* Parameter tuning suggestions
* Strategy creation
* Strategy-code generation
* Diagnostic parsing
* Result explanation

Agent styles include:

* Tool-using
* Structured-output
* Free-output
* Workflow-based

Guardrails:

* Agents do not have access to sensitive data.
* AI outputs can optionally be saved with backtest sessions for audit/review.

Potential future phrasing:

> Embedded lightweight LLM-wrapper agents into the trading platform to assist with strategy-code generation, diagnostic parsing, backtest interpretation, parameter tuning, and trading analysis.
