## Context

Soundbeats was a GameFi project focused on “gamifying music.” The core product concept allowed users to create their own Beat Saber-like courses for songs, combining music, gameplay, user-generated content, tokenized rewards, and NFT-based characters or avatars.

The project received some attention and investment during its early stage, despite being a relatively small GameFi prototype. The technical need was to connect a game client to a Sui blockchain backend in a secure and usable way.

---

## Role and Ownership

I served as the sole blockchain engineer on the early prototype.

I owned:

* Sui Move smart contract development
* Fungible token design for in-game points
* NFT contract design for characters/avatars
* Built-in Sui Move smart contract tests
* Secure NestJS REST API development
* Game-to-backend minting flow
* Authentication proof-of-concept
* Google OAuth proof-of-concept
* Smart wallet creation proof-of-concept
* Unity proof-of-concept for API communication and security

I worked loosely with a broader team of designers, game developers, and marketing contributors while providing the blockchain/backend foundation for the early game prototype.

---

## Blockchain / Smart Contract Work

Created Sui Move contracts for two major asset types:

1. **Fungible in-game token**

   * Used as points or rewards inside the game.
   * Functionally analogous to an ERC-20-style asset, but implemented using Sui Move and Sui-native design patterns.

2. **NFT assets**

   * Used for characters or avatars.
   * Functionally analogous to ERC-721-style NFTs, but implemented using Sui Move and Sui-native object ownership models.

The contracts included built-in on-chain tests using Sui Move’s native testing features and followed Sui blockchain best practices for package structure, ownership, minting, and object handling.

---

## Backend / API Work

Built a secure REST API using NestJS to allow the game code to communicate with the blockchain backend.

The API supported:

* Game-client authentication
* Token minting
* NFT minting
* Backend-controlled mint authorization
* Game-to-chain communication flows
* Secure access between the game client and blockchain minting logic

The API acted as the integration layer between the Unity/game client and the Sui blockchain contracts, preventing the game client from directly owning sensitive minting responsibilities.

---

## Unity / Game Integration Work

Learned enough Unity to create proof-of-concept integrations before a dedicated game developer took over.

Unity proof-of-concept work included:

* REST API communication from the game client
* Secure game-to-backend interaction flows
* Authentication flow testing
* Minting flow validation from game events
* Early integration patterns for later game development

This was not primarily a Unity developer role, but it demonstrated the ability to cross technical boundaries and validate the end-to-end gameplay-to-blockchain flow.

---

## Authentication / Smart Wallet Work

Created a proof-of-concept for authentication and smart wallet creation using Google OAuth.

This helped validate a smoother onboarding model for non-crypto-native game users, where users could authenticate through a familiar OAuth flow rather than manually managing blockchain wallet complexity from the beginning.
